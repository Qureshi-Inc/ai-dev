# Build SocialPlanner MVP — Phase 1: Set up foundation and authentication

Initialize the Next.js application with TypeScript and Tailwind CSS within the existing apps/socialplanner directory. Configure PostgreSQL with Prisma ORM including the User and Post models with all required fields, relationships, and indexes. Create and run the initial database migration.

Implement authentication using Auth.js (NextAuth) with a credentials provider. Build registration and login pages with secure password hashing using bcrypt. Add session management, logout functionality, and middleware that redirects unauthenticated users to the login page. Ensure all API routes check authentication status.

Create the Docker configuration including a Dockerfile, docker-compose.yml with web and PostgreSQL services, and .env.example with DATABASE_URL, AUTH_SECRET, and APP_URL. Add the GET /api/health endpoint that returns {"status": "ok"}. Set up the responsive application layout shell with navigation links for Dashboard, Calendar, All Posts, Create Post, and Log Out, including a collapsible mobile menu.
